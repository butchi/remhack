rem単位を用いたレスポンシブ対応ページの実装デモ

はじめに

このサイトがデモです。

ポイントは1つだけです。全てのサイズ指定にrem単位を用いることにより、メディアクエリ指定がhtmlに対するfont-sizeのみの変更で済むことです。

きっかけ

2015年9月ごろにAppleサイトのサポートページを開いたところ、大きな表示崩れが生じていました。

CSSを解読したところ、要素のところどころにrem単位を用いたサイズ指定が施されていました。

バグの原因（後述）を回避した上で、remを用いることでメリットがあるのではないかと思い、本記事を執筆しました。

rem単位とは

CSSには元来emという単位がありました。remはCSS3で導入され、HTML要素に指定されたフォントサイズが1remに相当します（remはroot emの略）。

バグの原因

Appleサイトでは、remのピクセル数として1pxが指定されていました。


html {
  font-size: 1px;
}

これにより、たとえば50remは50pxとして扱われるはずです。

FirefoxやInternet Explorerなどではその通りなのですが、Chromeではそうならないというのがポイントです。

Chromeの最小フォントサイズ問題

Chromeには、最小フォントサイズというブラウザー設定が存在し、デフォルトで10pxに指定されています。

ブラウザーの設定により、最小フォントサイズを下げることはできますが、それでも最低6pxまでしか下げられません。

1rem = 1pxということになるので、なぜAppleはpxを使わずremを使ったのかが謎です（後述のズームコントロールを視野に入れていたのでしょうかね？）。
